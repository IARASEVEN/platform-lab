# Architecture Decision Records

Every architecture decision in this repository gets an ADR: context, options
considered, decision, and consequences — including the bad ones. An ADR that
lists no downsides is marketing, not a record.

- Template: [`0000-template.md`](0000-template.md)
- Numbering is sequential and never reused.
- A superseded ADR is not deleted; its status changes to
  `Superseded by ADR-NNNN`.
